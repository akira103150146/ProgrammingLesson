## 4. 有符號 vs. 無符號比較陷阱

**觀念講解**  
- 當 `signed` 與 `unsigned` 互相比較時，`signed` 會先被轉成對應的 `unsigned`，負數變成極大值。  
- 這會導致看似「-1 > 1」卻回傳 `true` 的錯誤結果。

```cpp
#include <iostream>
using namespace std;

int main() {
    int a = -1;
    unsigned int b = 1;
    if (a > b)
        cout << "a > b\n";   // 事實上會進行 unsigned(-1) vs 1 → true
    else
        cout << "a <= b\n";
    return 0;
}
